## 출하 기능
1. **월간 LLM 인사이트 리포트** — Claude Haiku로 구조화 하이라이트(top_growth / anomaly / saving_tip) 생성. 온디맨드 + `monthly_insights` DB 캐시. 분류와 동일한 월간 예산 버킷 공유.
2. **입금/소득 분석** — `summary`에 income_total / net_savings / savings_rate 추가(이체 제외), `cashflow-by-month`(지출+수입 결합 추세), 대시보드 B+ 레이아웃.
3. **필수/비필수 토글** — 카테고리 기본 매핑(`essential.py`) + `essential_override` 컬럼(파생 계산), 거래 행 3-state 토글, `by-essential` 집계 도넛.

## DB 마이그레이션
- **0004** `add_essential_override_and_insights`
  - `transactions.essential_override BOOLEAN NULL` 추가
  - `monthly_insights (user_id, month, payload jsonb, generated_at)` 테이블 생성
- 자동 적용: API 컨테이너 기동 시 `alembic upgrade head` (apps/api/Dockerfile CMD). 별도 수동 작업 불필요.
- 백필 없음(파생 계산 모델). 기존 미사용 `essential`/`essential_reason` 컬럼은 그대로 방치.

## 신규/변경 API
| 메서드 | 경로 | 비고 |
|---|---|---|
| GET | `/insights?month=YYYY-MM` | 캐시된 리포트 또는 null |
| POST | `/insights/generate` (body `{month}`, `?force`) | 생성/캐시. 예산초과 503 BUDGET_EXCEEDED, LLM 실패 502 INSIGHT_GENERATION_FAILED |
| GET | `/dashboard/summary` | income_total / net_savings / savings_rate 필드 추가 |
| GET | `/dashboard/cashflow-by-month?last_n=` | 신규 (기존 `/dashboard/by-month` **제거**) |
| GET | `/dashboard/by-essential?month=` | 신규 |
| PATCH | `/transactions/{id}/essential` (body `{essential_override}`) | 신규 3-state |
| GET | `/transactions` | 응답에서 `essential`/`essential_reason` 제거 → `essential_override`/`effective_essential` |

> **Breaking**: `/dashboard/by-month` 제거 및 `TransactionOut` essential 필드 교체. 프론트엔드는 동일 PR에서 함께 갱신됨.

## 운영 사전 점검

- Redis 가동 필요 (예산 버킷). 운영은 기존 구성 유지.

## 알려진 한계
- 인사이트가 분류와 예산 버킷 공유 → 대량 생성 시 분류 예산 잠식 가능 (MVP 허용).
- 수입에 income/savings 카테고리 입금 포함(이체만 제외). 정교한 입금 구성 분석은 차후.
- `ESSENTIAL_DEFAULTS` 맵 변경은 코드 배포 필요(사용자별 설정 아님).
- essential 토글 낙관적 갱신 시 effective 표시가 재페치 전까지 약간 어긋날 수 있음.
- 웹 번들 ~1MB (Tremor + recharts) — W3부터의 기존 이슈, 코드 스플리팅은 별도 과제.
