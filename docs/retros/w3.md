## What worked
- **9 phase × 38 task 분해**가 정확히 들어맞음. Phase별 fresh subagent + 2단계 review가 매번 작은 버그 잡음 (KT 정규식 alias fragility, by_month GROUP BY alias, vitest hoisting, schemas import 위치, seed JSON shape, Tremor 색).
- **`effective_category = COALESCE(override, category)` 패턴**이 SELECT/필터/대시보드 집계 모두에서 일관 적용 — 단일 진실 출처.
- **TransactionRow 단일 타입**이 backend → lib/api.ts → 컴포넌트 모두 통과. 타입 충돌/리캐스트 0건.
- **subagent prompts에 spec 코드 그대로 인용** → implementer가 plan 파일 안 읽고도 정확 구현.
- gh CLI 자동 머지 → Phase 사이 latency 거의 0.
- 마이그레이션 0003 자동 적용 (deploy-api workflow의 entrypoint).

## What hurt
- **Tremor v3가 다크 모드를 자동으로 안 잡음** — `<html class="dark">` + `darkMode: "class"` 둘 다 명시해야 함. 차트 series 색도 default가 안 보여 `colors` prop 명시 필요. 시각 검수 후 hotfix 1회.
- **카드/통장 schema 충돌** — TransactionRow가 카드 거래 기준 (id/effective_category 필수)인데 seed JSON은 W1 시점 shape. guest.tsx 변경 + seed/routes.py에서 fallback fields 주입으로 해결. 미래에 다른 source가 추가되면 같은 문제 재발 가능.
- **CategoryChip dropdown이 `<details>` 카테고리 multi-select와 인접 시 outside click 충돌** — FilterBar `<details>`에 mousedown 리스너 직접 추가.
- vitest hoisting (`vi.hoisted`) — W2부터 반복 패턴이지만 매번 implementer가 처음 보고 막힘. 향후 web 컴포넌트 테스트 패턴 정리 필요.
- **사용자 데이터가 통장 중심**이라 dashboard 도넛이 비어 보임 가능성. 입금 거래는 amount < 0 → `amount > 0` 출금 필터에서 제외. W4에서 입금/소득 분석 추가하거나 사용자 데이터에 따른 빈 상태 메시지 보강 필요.
